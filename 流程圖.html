法規資料整理與 RAG 問答流程

主流程由上往下；左側 OCR 來源匯入 Markdown 精修區，右側 800200 baseline 從 Markdown 精修區分支。生成區與精修區分開，後續程式固定讀精修區。

  • 原始 / OCR 資料
  • 執行程式
  • 生成區
  • 精修區
  • JSON / Embedding
  • 問答展示

生成區與精修區分離

`03_Markdown生成區` 和 `06_CleanTree生成區` 是程式輸出；人工確認後，才把採用版本放進 `04_Markdown精修區` 或 `07_CleanTree精修區`。

OCR 也進 Markdown 精修區

未來 OCR 產出的 Markdown 進 `04_Markdown精修區`，讓 PDF 轉檔與 OCR 來源收斂到同一入口。

主線與 baseline

主線從 `07` 產生 structured JSON，再切 `all_nodes`、`leaf`、`table`、`hybrid`。`800200` 從 `04` 直接固定切段，用來比較檢索效果。
